# X-Intellect — новый сайт проекта

Новый сайт проекта **X-Intellect** (ранее — «Сфера Разума», основан в 2012 г. А. Г. Глазом):
Laravel-приложение с админкой, публичной частью на архивных мотивах, аудиоплеером,
глоссарием с тултипами и SEO-автоматизацией. Цель проекта — сохранить материалы
архива сайта (статьи, вики, аудиозаписи, форум) в удобном для чтения виде.

Сейчас в базе: 218 страниц, 78 терминов глоссария, 73 аудиозаписи,
архив форума (113 тем / 1551 сообщение), 724 редиректа со старых адресов.

## Стек

- **Laravel 13** (PHP 8.3), Blade — серверный рендер, SEO из коробки
- **SQLite** локально (`database/database.sqlite`, не в git — см. «Материалы» ниже), MySQL на проде
- **Alpine.js** — аудиоплеер, тултипы глоссария, фильтры; **Trix** — редактор админки; **Vite + Tailwind** — сборка фронтенда
- **Database-очередь + cron** (`queue:work --stop-when-empty`) — фоновые задачи без воркер-процесса

## Особенности нового сайта

Старый сайт (2012–2017) жил на трёх отдельных движках — WordPress, MediaWiki
и phpBB — с flash-плеером и только ПК-версией. Новый сайт собран заново
и даёт то, чего на старом не было:

### Контент и чтение

- 📚 **Единая платформа.** Статьи, вики, глоссарий, аудио и архив форума —
  в одной системе с общим дизайном и навигацией вместо трёх отдельных движков.
- 📖 **Новый улучшенный глоссарий.** У каждого термина — собственный
  индексируемый адрес (`/glossary?term=…`), живой фильтр без перезагрузки
  страницы, алфавитный указатель, копирование термина или ссылки в один клик,
  форматирование определений и переход к связанной статье «Подробнее».
- ✨ **Подсветка терминов в материалах.** Термины глоссария автоматически
  распознаются в тексте каждой статьи: наведите курсор — и всплывёт подсказка
  с определением, без перехода на другую страницу.
- 🕰 **Ссылка на архивную копию в каждом материале.** Рядом с заголовком —
  бейдж эпохи («Сфера Разума», «Архив X-Intellect», «Архив вики», «Новый
  материал») и ссылка на оригинал в Wayback Machine; у страниц ведётся
  история изменений (ревизии).
- 🎧 **HTML5-аудиоплеер вместо flash.** Плейлисты, скорость воспроизведения,
  перемотка ±15 с, запоминание позиции прослушивания; плеер вставляется
  в текст short-кодом `[[audio:ID]]`.
- 💬 **Архив форума phpBB** в режиме чтения — темы и сообщения восстановлены
  из офлайн-копий и Wayback Machine.

### Навигация и поиск

- 🔍 **Улучшенный поиск.** Один поиск по всему архиву — статьям, вики и
  стенограммам сеансов; регистр не важен (в том числе для кириллицы),
  в выдаче — и страницы, и термины глоссария с быстрым переходом.
- 🧭 **Улучшенная навигация.** Разделы с подразделами и стабильными
  адресами страниц, выпадающие подменю в шапке, хлебные крошки на каждой
  странице, боковое меню вики, сворачиваемые описания «О разделе»,
  дозагрузка списков «Показать ещё» без перезагрузки.
- 🔗 **Блок «Поделиться»** под каждым материалом: ВКонтакте, Одноклассники,
  Telegram, MAX, LiveJournal, Мой Мир и копирование ссылки — приватно,
  без внешних SDK и трекеров.

### Оформление и техника

- 📱 **Адаптивность.** Старый сайт имел только ПК-версию — новый корректно
  выглядит на смартфоне, планшете и десктопе: меню-бургер, резиновые
  таблицы, плитки и плеер.
- 🌗 **Светлая и тёмная темы** с «космическим» оформлением: звёздное небо,
  стеклянные панели (глассморфизм), выбор темы запоминается.
- 🚀 **SEO-автоматизация.** JSON-LD (Article, FAQPage, DefinedTerm,
  BreadcrumbList), sitemap, canonical, 724 постоянных редиректа со всех
  старых адресов WordPress и MediaWiki, `robots.txt` + `llms.txt`.
- 🔒 **Приватность.** Сайт не собирает данных посетителей: ни аналитики,
  ни трекеров, ни сторонних скриптов.
- 🛠 **Современная админка.** Trix-редактор с модальным редактором таблиц,
  выравниванием картинок с обтеканием, загрузкой картинок/аудио/PDF до 170 МБ
  прямо в текст, редакторами описаний разделов и определений глоссария,
  ролями admin/editor; редакторы видят черновики на сайте до публикации.
- ⚙️ **Автоматизации контента.** Авто-slug транслитерацией, автопростановка
  тултипов глоссария при сохранении, внутренние ссылки автоматически
  приводятся к относительным (вставленный `localhost` не попадёт на прод),
  адреса терминов стабильны при переименовании.

## Что реализовано

| Раздел | Реализация |
|---|---|
| Модель данных | `sections` (иерархия `parent_id`), `pages` (seo JSON, `source_type` — эпохи контента, ревизии), `media`, `glossary_terms`, `redirects`, `menu_items`, `forum_topics`/`forum_posts` |
| Admin CMS | `/admin` (auth + роли admin/editor, rate-limit логина, noindex), CRUD всего, Trix (таблицы, картинки, аудио `[[audio:ID]]`, относительные ссылки) |
| Публичная часть | космическая тема (светлая/тёмная), стеклянные плитки и панели, бейджи эпох, хлебные крошки, вики-сайдбар, таймлайн истории, дисклеймер в футере |
| Поиск | `/search` — регистронезависимый (FULLTEXT на MySQL, mb-фолбэк на SQLite), включая unlisted-стенограммы; блок терминов глоссария в выдаче |
| Аудиоплеер | Alpine-компонент: плейлист, скорость, перемотка ±15с, позиция в localStorage; short-код `[[audio:ID]]` |
| Глоссарий | `/glossary` + адрес термина `?term=slug`, живой фильтр, копирование, разметка в определениях, автотултипы в статьях |
| Форум | `/forum` — архив phpBB в режиме чтения |
| SEO | авто-slug (транслитерация), meta из seo-полей, JSON-LD (Article/Person/FAQPage/DefinedTerm/BreadcrumbList), `sitemap:generate`, middleware редиректов (301 архивные / 302 `/go/*`, кеш 1 час) |
| robots/llms | статичные `public/robots.txt`, `public/llms.txt` |
| Импорт архива | офлайн-слепок + Wayback Machine: основной сайт, вики, термины, аудио, форум, перелинковка, аудит полноты (`docs/audit/`) |

## Локальная разработка

```bash
composer install
cp .env.example .env && php artisan key:generate
touch database/database.sqlite
php artisan migrate --seed                          # админ: SEED_ADMIN_EMAIL / SEED_ADMIN_PASSWORD в .env
php artisan storage:link
npm install && npm run dev                          # или npm run build
php -S 127.0.0.1:8753 dev-server.php                # роутер с HTTP Range — перемотка аудио работает
```

(`php artisan serve` тоже подходит, но без Range-запросов перемотка
аудиоплеера будет сбрасывать позицию.)

Пароль администратора задаётся через `.env` (`SEED_ADMIN_EMAIL`/`SEED_ADMIN_PASSWORD`)
и меняется после первого входа на `/profile`.

Тесты: `php artisan test` (126 тестов: публичные страницы, поиск, редиректы, роли, SEO, ревизии, форум, редакторы).

База данных (`database/database.sqlite`) намеренно **не в git**: публиковать
БД «как есть» небезопасно. Вместо этого сайт заполняется **импортом из архива**, а не готовой БД:

**https://cloud.mail.ru/public/sUfY/jpDWsSH5N** - файлы с старого сайта
**https://disk.yandex.ru/d/E9uI6yzVYZngaQ** - файлы курсов продублировал на Яндекс Диск

В облаке лежит полный офлайн-слепок сайта — все статьи, вики-страницы, описания
проектов и аудиозаписи (отдельными файлами слишком велики для git).
Скачав архив, сайт легко наполняется командами из раздела «Импорт архива» ниже —
за несколько прогонов миграции/сидеров + импортёров восстанавливается вся база. 
Есть еще 2 варианта импорта файлов: вручную разложить по папкам или 
попросить ИИ-агента сделать это за вас, я пользовался Claude.

Картинки статей (`storage/app/public/media/archive/`, ~17 МБ) — небольшие и
безопасные для публикации, они в этом репозитории.

## Импорт архива

Команды разбора локального офлайн-слепка сайта (`app/Console/Commands/`), в порядке прогона:

- `site:structure-2026 --pre` — подготовка структуры до импорта (типы источников)
- `import:offline-explorer {archive}` — основной сайт → страницы разделов (черновики)
- `import:offline-wiki {wiki-dir}` — вики → страницы «Вики» + термины в «Глоссарий»
- `import:offline-audio {wiki-dir}` — привязка аудио к сеансам, консультациям и «Приветствию»
- `import:offline-forum {forum-dir}` — архив форума phpBB (темы + сообщения, только чтение)
- `import:wayback-wiki --from --to [--snapshot={archive}]` — доимпорт вики-страниц из
  Wayback Machine (чего нет в слепке). `--snapshot` обязателен, если нужны картинки:
  снимки ссылаются на них корне-относительными путями MediaWiki, файлы берутся из слепка
- `remap:archive-links` — перелинковка внутренних ссылок на актуальные адреса нового сайта
- `links:restore {archive} [--published-only]` — точечно возвращает ссылки, срезанные
  при импорте (в теле остался голый текст). Вставка хирургическая: за пределами
  вставленного `<a>` тело не меняется, поэтому команда безопасна и для опубликованного.
  Прогонять ПОСЛЕ импортёров и `remap` — цели ссылок должны уже существовать
- `site:structure-2026` — раскладка страниц по подразделам
- `content:backfill` — архивные даты (из Wayback-меток) и анонсы для пустых полей
- `site:content-fixes-2026 [--snapshot={archive}]` — правки контента: чистка служебных
  строк таблиц, сводные таблицы, ссылки на странице «Техники», починка редиректов,
  относительные ссылки. С `--snapshot` (путь до `…/www.x-intellect.org`) странице
  «Картины Учителей Ноосферы» возвращаются 23 картины: сама страница пришла из
  Wayback, а он отдаёт картинки внешними ссылками — файлы берутся из офлайн-слепка
- `content:sync-dates {archive} --cdx` — сверка `published_at` с датами добавления на
  старом сайте (архивы WordPress, подвалы вики, даты в заголовках сеансов; `--cdx` —
  первый снимок Wayback для остального). При 429/503 от CDX просто повторить —
  ответы кешируются в `storage/app/wayback-cdx-dates.json`
- `sessions:restore-missing {archive}` — создаёт страницы сеансов, утраченных всюду
  (нет ни в слепке, ни в Wayback), из аудиозаписей в папках архива; берёт список из
  тел навигационных страниц «Сеансы …»
- `import:sferarazuma` — материалы проекта «Сфера Разума» (до 2012, вёл А. Г. Глаз):
  аудио из папок архива + описания из веб-архива вики SphereWiki (см. ниже)
- `sessions:enrich-sfera` — дополняет заглушки сеансов стенограммами из «Сферы Разума»
- `site:menu-subsections` — подразделы в выпадающие меню шапки
- `audit:archive {archive} --wayback --fix-redirects` — аудит полноты импорта → отчёты `docs/audit/`
- `sitemap:generate` — карта сайта

Все команды идемпотентны и поддерживают `--dry` (просмотр без записи) и,
где применимо, `--refresh` (перечистка тела уже импортированных страниц).
Импортированные страницы создаются черновиками — публикация вручную после вычитки.

**Опубликованное не перезаписывается.** `--refresh` пропускает страницы со статусом
`published` и страницы с ручными правками в истории (гард в коде, тесты —
`tests/Feature/PublishedPagesProtectedTest.php`). Единственные правки, разрешённые
опубликованному: вставка потерянной ссылки (`links:restore`) и удаление служебной
строки таблицы «Аудио Запись» (`site:content-fixes-2026`) — обе точечные, с ревизией.

Если прогоняется `--refresh`, порядок важен: он возвращает тело к исходнику, поэтому
после него нужно повторить `remap:archive-links`, `site:content-fixes-2026` и
`links:restore` — иначе вернутся служебные строки таблиц, а ссылки снова пропадут.

Перечисление файлов слепка — только через `App\Services\OfflineSnapshotIndex`:
Offline Explorer раскладывает часть страниц по подпапкам `%&Ovr0…%&Ovr119`, обрезает
длинные имена файлов (дописывая хэш) и сохраняет рядом просмотры diff и старых
ревизий. Поэтому вид страницы определяется по содержимому, а не по имени файла.

Аудио: `import:offline-audio` ищет mp3 по ссылкам в статье и по дате в заголовке —
в слепке и папках из `config/archive.php` (`--audio-dir` переопределяет). Среди копий
одной записи побеждает самая длинная: в слепке часть записей обрезана (у 20081111 —
22 мин против 74 мин в «Сайт и файлы + аудио»).

### Материалы проекта «Сфера Разума» (до 2012)

До 2012 года проект существовал под именем «Сфера Разума» (`sferarazuma.ru`), его вёл
А. Г. Глаз. Аудиозаписи тех ченнелингов лежат в архиве Николая Морозова
(`config/archive.php` → `sfera_audio_dirs`: папки «Аудиозаписи контактов/Новое» и
«Аудио контакты 93-94»), а стенограммы и метаданные — в веб-архиве вики SphereWiki
(категория «Стенограммы», снимок не позднее конца 2012 — раньше страниц там нет).

`import:sferarazuma` для каждой записи, которой ещё нет на сайте, берёт описание со
страницы SphereWiki (сервис `App\Services\SferaRazumaArchive`, кеш в
`storage/app/sferarazuma/`) и создаёт черновик с `source_type=archive_sferarazuma`
(бейдж «Сфера Разума (до 2012)»). Если стенограммы в архиве нет — заглушка с
максимумом данных из имени файла. Ссылки на `sferarazuma.ru` не сохраняются: mp3
заменяет плеер, прочие разворачиваются в текст. `sessions:enrich-sfera` дополняет
уже созданные сеансы-заглушки полными стенограммами из «Сферы Разума».

Особенности: идемпотентность по ключу-дате (иначе повторный прогон плодит дубли —
`alreadyOnSite` намеренно исключает свой тип, чтобы допривязывать аудио); аудио
берётся по прямому пути файла, а не только через `AudioLibrary::byDateKey` (та
индексирует только mp3 и матчит «20…» в имени, а тут бывают `.wav` и файлы «93-94»
с именем вида `24-12-93` при ключе `19931224`).

## Безопасность

- `.env` не в git (секреты — `APP_KEY`, пароли, токены)
- Rate-limit на `/login`, публичная регистрация отключена
- `composer audit` / `npm audit` перед релизом
- Загрузка файлов — по белому списку MIME (`App\Models\Media::MIMETYPES`, общий для раздела
  «Медиа» и редактора Trix). SVG и HTML запрещены намеренно: медиа отдаётся с `/storage` того же
  происхождения, что и сайт, а имя в хранилище получает расширение по настоящему MIME-типу —
  такой файл исполнил бы скрипты в контексте домена
- Защитные заголовки — middleware `SecurityHeaders` (`nosniff`, `X-Frame-Options`,
  `Referrer-Policy`; HSTS только по HTTPS). CSP не ставится: на страницах инлайн-скрипты
  (JSON-LD, Alpine). Файлы `/storage` идут мимо PHP — заголовки для них в `storage/app/public/.htaccess`
- Роль (`role`) вне mass assignment: назначается явно (сидер, tinker), иначе любое
  `update($request->all())` стало бы эскалацией до `admin`
- Роль `editor` — только доверенным: тело страницы рендерится без санитизации (`{!! $body !!}`),
  так что редактор через разметку дотягивается до действий администратора
- Проверки перед деплоем и после — чек-лист Этапа 8 плана (`APP_DEBUG=false`,
  `SESSION_SECURE_COOKIE=true`, свой `APP_KEY`, смена пароля сид-админа)
- Тесты: `tests/Feature/SecurityTest.php`
