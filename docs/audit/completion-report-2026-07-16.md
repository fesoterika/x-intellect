# Отчёт о доукомплектовании архива — 16.07.2026

Что сделано по задаче «наиболее полный архив»: доимпорт пропавших страниц, аудио,
восстановление ссылок, удаление служебных строк таблиц. Опубликованные страницы
не перезаписывались (см. раздел «Сохранность ручной работы»).

## Причина недостачи (найдена и устранена)

Все импортёры и аудит перечисляли только корень `wiki/` слепка
(`File::glob($base.'/index.php@title=*')`) и отбрасывали файлы с `&` в имени.
Offline Explorer же:

1. ссыпает часть страниц в подпапки `%&Ovr0…%&Ovr119` — они были невидимы
   (пример: «Сеанс с силами 20070730b» лежал в `%&Ovr3/`);
2. **обрезает длинные имена файлов и дописывает хэш** — поэтому фильтр «нет `&`
   в имени» не работал: под видом статей проходили просмотры diff и старых ревизий,
   а настоящие статьи терялись. У одной страницы бывает до 344 файлов-вариантов,
   из которых канонический — ровно один;
3. по той же причине обрезаны и `href` внутри страниц (`План трени…A04AB161`),
   а файлы-цели лежат в чужих папках — ссылки не резолвились.

Решение — сервис `App\Services\OfflineSnapshotIndex`: обходит корень и все `%&OvrN`,
определяет вид страницы **по содержимому** (`mw.config` + разметка diff/ревизии/
редиректа), даёт карту «имя файла → путь» для резолва ссылок. На него переведены
`import:offline-wiki`, `import:offline-audio`, `audit:archive`, `links:restore`.

Честная картина слепка: **263 содержательные статьи** (было «видно» 172).
