# Импорт архива: дубли страниц и обязательная ручная проверка

> Вынесено из README (раздел «Импорт архива»).

**При импорте архива могут появляться дубли страниц.**

Источников несколько, и один и тот же материал в них может лежать под разными
адресами и именами: офлайн-слепок 2015 года (Offline Explorer с подпапками
`%&Ovr0…%&Ovr119` и обрезанными именами файлов), снимки Wayback Machine
(основной сайт и вики), архив «Сфера Разума». Импортёры идемпотентны по своим
ключам (slug, дата записи), но при расхождении ключей между источниками одна и
та же страница может быть создана повторно.

## Что делать

**Необходима полная ручная проверка после всех прогонов импортёров:**

1. Прогнать все команды импорта в порядке из README (раздел «Импорт архива»).
2. Прогнать аудит полноты: `php artisan audit:archive {слепок} --wayback --fix-redirects`
   — отчёты складываются в `docs/audit/`.
3. Вручную просмотреть списки страниц в админке (`/admin`) по каждому разделу:
   искать одинаковые или почти одинаковые заголовки, страницы-огрызки,
   повторяющиеся сеансы с разными slug.
4. Найденные дубли: у лишней страницы снять публикацию или удалить её,
   при необходимости добавить редирект со «второго» адреса на основной.
5. Проверку повторять после каждого нового прогона импортёров
   (в том числе после `--refresh` и доимпорта из Wayback).

Импортированные страницы создаются черновиками, публикация — вручную после
вычитки, поэтому дубль не попадёт на сайт, пока его не опубликовали. Но в
админке и в аудитах дубли засоряют картину — вычищать их нужно обязательно.
